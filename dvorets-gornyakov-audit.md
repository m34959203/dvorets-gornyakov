## 1. GitHub-репозиторий

### Стек
- Next.js **16.2.2** (App Router) + React **19.2.4** + TypeScript 5 + Tailwind CSS 4
- PostgreSQL 16 через `pg` — **raw SQL, без ORM**
- Auth: JWT (`jose` 6) + Argon2 + RBAC (admin/editor)
- AI-чат: Google Gemini (`gemini-2.0-flash`, free tier)
- Валидация: Zod 4
- i18n: своя реализация с префиксами `/kk/` и `/ru/`
- Деплой: Docker Compose + Caddy
- Состояние: 32 коммита, 1 контрибьютор, 0 stars/forks, ветка `master`

### Структура
```
src/app/[locale]/(public)/   публичные страницы
src/app/[locale]/(admin)/    админка с сайдбаром
src/app/api/                 REST endpoints
src/components/ui|features/  UI-кит и фичи
src/lib/                     auth, db, gemini
sql/                         нумерованные миграции (001_, 002_...)
caddy/  docker-compose.yml   инфраструктура
docs/  scripts/  public/     документация, утилиты, статика
```

### Сильные стороны
- Современный стек (Next 16, React 19, Tailwind 4) на день написания отчёта.
- Безопасность описана и реализована: параметризованные SQL, JWT в httpOnly+secure+sameSite cookie, Argon2 для паролей, CSP-заголовки в Caddy, Zod-валидация, rate-limit 3/час/IP на enrollment.
- Чёткая bilingual-конвенция: поля БД `field_kk`/`field_ru` + helper `getLocalizedField`, UI-строки в `src/messages/{kk,ru}.json`.
- Conventional commits, запрет на коммит `.env`, feature branches → PR в main.
- `CLAUDE.md` (developer guide, 94 строки) — отличный онбординг.

### Что улучшить
1. **README.md = шаблон `create-next-app`**. Перенесите содержимое `CLAUDE.md` (или дублируйте) в README — внешний читатель не понимает, что за проект.
2. **Расхождение версий**: `package.json` ставит Next 16.2.2, а `CLAUDE.md` пишет «Next.js 15». Обновить документацию.
3. **Нет тестов**: `scripts` содержит только `dev/build/start/lint`. Добавьте Vitest для unit и Playwright для e2e — минимум для критичных потоков (auth, enrollment, i18n-роутинг).
4. **Нет CI/CD**: в `.github/workflows/` пусто. Минимум — workflow `lint + build + typecheck` на PR.
5. **Нет pre-commit**: добавить `husky` + `lint-staged` + `prettier`, чтобы исключить грязные коммиты.
6. **Raw SQL без ORM** даёт скорость, но дороже в поддержке. Рассмотрите тонкий `Kysely` или `Drizzle` для типобезопасности запросов без потери контроля.
7. **`instrumentation.ts`** есть, но без OpenTelemetry/Sentry в зависимостях — настройте сбор ошибок (Sentry/Logflare) для прод.
8. **Нет лицензии** (`LICENSE` файл). Для государственного учреждения важно явно указать (GPL/MIT/Proprietary).
9. **Нет `SECURITY.md`** и шаблонов issue/PR.
10. **Аргон2 как native-модуль** на Alpine-Docker может ломать сборку — проверить, что Dockerfile использует `node:20-bookworm-slim` или ставит `python3 make g++`.

## 2. UX/UI-аудит сайта

URL: `https://tracy-wal-sep-uses.trycloudflare.com` (preview через cloudflare-туннель).
Проверены страницы: `/kk`, `/ru`, `/ru/events`, `/ru/clubs`, `/ru/rent`, `/ru/contacts`.
Чат-бот протестирован живым запросом.

### Что хорошо
- **Айдентика**: тёплый бежевый + изумрудно-зелёный + золотой — фирменно, культурно «своё», уместно для дворца культуры.
- **Навигация**: 8 пунктов, понятная иерархия (Главная / Новости / Мероприятия / Кружки / Аренда залов / О нас / Ресурсы / Контакты), активный пункт подсвечен.
- **Bilingual**: переключатель KZ/RU в шапке меняет `htmlLang`, URL и весь контент. Канонические URL вида `/kk/...` и `/ru/...`.
- **Семантика**: корректная иерархия H1→H2→H3, есть `<main>` и `<footer>`, `aria-label` на иконочных кнопках («Open menu», «Chat»).
- **Доступность форм**: 0 input-ов без подписи на главной (по DOM-проверке).
- **Календарь** на главной с подсвеченными датами событий — сильный UX-приём.
- **Карточки события**: дата крупным акцентом (22, 15, 01), время, место, CTA.
- **Кружки**: фильтры «направление / возрастная группа» + AI-тест «Какой кружок вам подходит?» — отличная вовлекающая фича, на странице видны реальные фото.
- **Аренда залов**: hero с фотографией ансамбля, 3 карточки залов с `alt` («Большой зал, сцена», «Камерный зал», «Репетиционный зал»), CTA «Оставить заявку».
- **Контакты**: адрес, два телефона (приёмная / касса), email, часы работы, форма обратной связи, плейсхолдер карты.
- **Чат-виджет** работает: открывается, есть текстовое поле и микрофон (голосовой ввод).

### Критичные замечания

**1. Hero выглядит как недогруженная страница.**
Орнамент-«стена кирпичей» под заголовком визуально воспринимается как skeleton-loader. Пользователь думает, что страница не догрузилась. Сделать орнамент тоньше / с меньшей плотностью / в виде штриховки.

**2. Нет настоящих фотографий на главной.**
Все карточки событий и новостей — градиентные плейсхолдеры. Для культурного учреждения это критично: сайт не вызывает желания прийти. Срочно загрузить фото из архива дворца.

**3. CTA не соответствует контенту.**
Все события помечены «Тегін / Бесплатно», но кнопка — «Купить билет / Билет сатып алу». Должно быть «Зарегистрироваться / Тіркелу» или «Записаться».

**4. Дублирование контента.**
«Наурыз концерті», «Би шеберханасы», «Жас суретшілер көрмесі» появляются и в «Айдағы іс-шаралар», и в ««Арман» алдағы іс-шаралары». Это или баг seed-данных, или плохое разделение секций.

**5. Подозрительные данные.**
Выставка «04:00 · Галерея» — 4 утра. Похоже, в БД хранится UTC без приведения к Asia/Almaty или это демо-данные. Проверить.

**6. Чат-бот не отвечает по делу.**
На вопрос «Какие кружки есть для детей 7-10 лет?» ответ: «Через сайт или администрацию. Телефон: +7 (7102) 77-77-77». Это generic fallback — либо не подключен `GEMINI_API_KEY`, либо системный промт не передаёт контекст о кружках. Также нет suggested questions при открытии — пользователь не понимает scope.

### WCAG / контраст
- `САТПАЕВ ҚАЛАСЫ · 1974 ЖЫЛДАН БЕРІ` — тонкий жёлтый на зелёном, шрифт мелкий: вероятно ниже 4.5:1.
- Подзаголовки времени/места в карточках («12:00 · Негізгі зал») — серый по тёмно-зелёному, контраст слабый.
- Дни недели в календаре (Дс/Сс/Ср...) почти невидимы.
- Индикатор пагинации «01 / 1» блёклый.
- Стат-блок «Наша история» (22 / 758 / 60+ / 3) — крупные цифры, но подписи под ними мелкие и серые на бежевом.

**Рекомендация**: пройтись по дизайну с axe DevTools / Lighthouse, поднять контраст вторичного текста до минимум 4.5:1 (для обычного) и 3:1 (для крупного).

### SEO и meta
- **Нет** `og:title`, `og:description`, `og:image` — ссылка на сайт в Telegram/WhatsApp/VK будет голой.
- **Нет** `theme-color` (важно для мобильного браузера).
- **Нет** JSON-LD Schema.org: добавить `Organization`, `LocalBusiness`, `Event` (особенно для афиш — это даёт rich snippets в Google).
- Проверить наличие `sitemap.xml`, `robots.txt` (на превью не виден).
- Title-страницы одинаковые на разных URL — добавлять контекст («Контакты — Дворец горняков»).

### A11y
- Нет `skip-link` к `<main>` — для клавиатурных пользователей это базовое требование.
- Видимый `:focus`-индикатор не оценивался — желательно проверить вручную табом по странице.
- Кнопки каруселей `‹` `›` без `aria-label` — добавить «Предыдущий слайд / Следующий слайд».

### Адаптивность
Полноценный мобильный аудит не получилось провести (resize окна не меняет viewport браузера ниже ~1500px на тестовом стенде). Нужно проверить вручную в DevTools (375×667 / 390×844 / 414×896 / 768×1024).

### Производительность
- 22 inline SVG на главной — норма.
- Скриншоты дважды таймаутили (CDP > 30s) — возможны тяжёлые CSS-анимации или большой initial JS bundle. Прогнать Lighthouse и проверить INP/LCP.
- Cloudflare-tunnel добавляет латентность; не показатель прода.

### Безопасность фронтенда
- HTTPS — ✓.
- Чат-виджет шлёт запросы — проверить, что нет утечки токена в URL.
- Нет cookies/privacy-баннера. Для КЗ обязательность спорна, но рекомендуется (особенно если используется аналитика).

## 3. Приоритезированный план улучшений

### P0 (срочно, до публичного запуска)
1. Загрузить настоящие фотографии (события, новости, главная).
2. Починить чат-бот: проверить `GEMINI_API_KEY`, дать системному промту контекст кружков/афиш, добавить 3-4 suggested questions.
3. Исправить CTA «Купить билет» на «Записаться» для бесплатных событий.
4. Исправить таймзону событий (Asia/Almaty) и удалить дубли seed-данных.
5. Добавить `og:image`, `og:title`, `og:description`.

### P1 (в первый месяц)
6. Поднять контраст вторичного текста до WCAG AA.
7. Smooth-орнамент в hero (или его прозрачность).
8. Skip-link, aria-label у каруселей, видимый focus.
9. JSON-LD для `Event`, `Organization`.
10. Проверить мобильную верстку и зафиксировать баги.

### P2 (улучшения и масштабирование)
11. Перенести `CLAUDE.md` → расширенный `README.md`.
12. CI (lint+typecheck+build), pre-commit (husky+lint-staged), Vitest+Playwright.
13. Sentry / OpenTelemetry для prod-наблюдаемости.
14. Возможно — Kysely/Drizzle для типобезопасных SQL.
15. Цветовая система карточек по типу события (концерт / выставка / мастер-класс) вместо случайного зелёный/золотой.

## Итог
Проект на правильном пути: грамотный стек, security-aware архитектура, чёткий bilingual-подход, заложен AI-чат. Перед публичным запуском критичны **контент** (фото, реальные данные, рабочий чат-бот) и **WCAG-контраст**. Технически репозиторий требует CI, тестов и нормального README.
