# Бот бронирования залов (WhatsApp)

Диалоговый бот ведёт пользователя по шагам (язык → зал → дата → время → гости →
формат → имя → телефон → email → подтверждение) и создаёт заявку в `rental_requests`
(та же таблица, что и веб-форма) + шлёт уведомление оператору в Telegram-канал.

> Telegram-канал бота убран — бронирование идёт только через WhatsApp.
> (Telegram остаётся для уведомлений оператору и автопостинга новостей/событий.)

## Архитектура

- FSM-движок: `src/lib/rent/bot/engine.ts` (чистый, RU/KK).
- Состояние диалога: таблица `bot_sessions` (миграция `sql/013_bot_sessions.sql`).
- Оркестратор: `src/lib/rent/bot/handle.ts` (`handleInbound("whatsapp", chatId, text)`).
- Создание заявки: `src/lib/rent/create.ts` (`createRentalRequest`, общая с веб-формой).
- WhatsApp-транспорт: встроенный Baileys-сокет `src/lib/wa/runtime.ts` (отвечает
  напрямую). Сессия в БД (`wa_auth`, миграция `sql/014`, `useDbAuthState`).
- Кнопка на сайте: `rent/[slug]` → диплинк `wa.me/<номер>?text=/start rent_<slug>`
  (`src/lib/rent/bot/links.ts`). Пока номер не привязан — показывается веб-форма (фолбэк).

## Активация (WhatsApp)

Публичный домен НЕ нужен — Baileys держит исходящий сокет к web.whatsapp.com.

1. Применить миграции: `sql/013_bot_sessions.sql`, `sql/014_wa_auth.sql`.
2. Войти в админку → раздел **WhatsApp** (`/admin/wa`, только admin).
3. Нажать «Подключить» → отсканировать QR телефоном Дворца
   (WhatsApp → Связанные устройства → Привязка устройства).
4. После статуса «Подключён» номер автозаполняет `site_settings.whatsapp_phone`,
   и на странице зала появляется кнопка «Забронировать в WhatsApp».

Сессия хранится в БД и восстанавливается после рестарта (авто-resume в
`instrumentation.ts`). Отключить WA-мост на старте: `WA_DISABLED=1`.

## Опциональный внешний WA-гейт

Если вместо встроенного сокета используется внешний Baileys-гейт: вебхук
`POST /api/rent/bot/whatsapp` (тело `{ from, text }`, секрет в `WA_WEBHOOK_SECRET`),
исходящие — `WA_GATEWAY_SEND_URL` (+ `WA_GATEWAY_TOKEN`). По умолчанию не используется.

## Поведение

- `/start` (или `/start rent_<slug>` из диплинка) — начинает/сбрасывает диалог;
  при payload зал предвыбран, шаг зала пропускается.
- `/cancel`, «отмена» — сброс.
- Варианты отправляются нумерованным списком — пользователь отвечает цифрой.
