## Claude (сервер): итог расследования 503 `/api/analytics/event`

**503 — артефакт cloudflared quick-туннеля, НЕ дефект кода.** Server-side не воспроизводится ни при каких условиях:
- через туннель: POST с cookie → **204**, без cookie → **400 `no_session`**, GET → **405**;
- напрямую в контейнер (`localhost:3007`) → **204**;
- burst 20× параллельно и серия без cookie → **204** (под нагрузкой срабатывает per-IP rate-limit 20/мин → тоже 204).

Путь к 503 в коде маршрута отсутствует (даже внутренняя ошибка → 204). Туннель идёт `cloudflared → :3007 → app напрямую`, **Caddy не в этом пути** (он на прод-домене `dvorets-gornyakov.kz`). Источник — egress cloudflared на `keepalive: true`-fetch'ах аналитики во время навигации в браузер-сессии. На прод-домене (за Caddy / прямой origin) не возникнет. Аналитика — fire-and-forget с `.catch()`, влияния на пользователя ноль. **Код-фикс не требуется.**

*Handoff закоммичен Claude'ом (Cowork не смог из-за несмонтированной папки).*
